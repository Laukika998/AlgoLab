Searching in Array:

Searching in an array is a fundamental operation is used to find the location of a particular value or element within the array. Several algorithms are commonly employed for searching in arrays, each with its own characteristics and use cases.

1.Linear Search:

Linear search, also known as sequential search, is a simple searching algorithm that iterates through each element in the array one by one until the target element is found or the end of the array is reached.

Algorithm:

1.Start from the first element of the array.
2.Compare each element with the target element.
3.If a match is found, return the index; otherwise, continue to the next element.
4.Repeat until the target element is found or the end of the array is reached.

2.Binary Search:

Binary search is an efficient searching algorithm designed for sorted arrays. It works by repeatedly dividing the search interval in half.

Algorithm:

1.Start with the entire array.
2.Compare the target element with the middle element of the array.
3.If they are equal, the search is complete, and the index is returned.
4.If the target is less than the middle element, repeat the search on the left half.
5.If the target is greater, repeat the search on the right half.
6.Continue dividing and searching until the target is found or the search interval is empty.

3.Hashing (Hash Table) Search:

Hashing is a technique that involves mapping data to a fixed-size array, known as a hash table. It is suitable for scenarios where quick data retrieval is required.

Algorithm:

1.Use a hash function to map the target key to an index in the hash table.
2.Handle collisions (cases where two keys map to the same index) using techniques like linear probing or chaining.
3.Search for the target key in the corresponding index.
4.If found, return the associated value; otherwise, handle collisions and continue the search.

Use Cases:

Linear search is suitable for small arrays or unsorted data.
Binary search is efficient for large sorted arrays but requires the array to be sorted.
Hashing is useful for quick data retrieval and is commonly used in database indexing and symbol tables.
